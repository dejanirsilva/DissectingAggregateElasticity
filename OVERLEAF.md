# Overleaf Integration (Paper-Only Sync)

This repository includes code, notes, and literature files that should not be sent to Overleaf.
Use the workflow below so Overleaf receives only the `paper/` project.

## 1) One-time setup

1. In Overleaf: open your project, then `Menu -> Git`.
2. Copy the Git URL (looks like `https://git.overleaf.com/<project-id>`).
3. In this repo, add the Overleaf remote:

```bash
git remote add overleaf https://git.overleaf.com/<project-id>
```

4. Make the helper scripts executable:

```bash
chmod +x utils/scripts/push_overleaf.sh
chmod +x utils/scripts/pull_overleaf.sh
```

## 2) Pull the current Overleaf project into this repo

Because the paper currently lives in Overleaf, start by importing it:

```bash
utils/scripts/pull_overleaf.sh
```

Defaults:
- Remote: `overleaf`
- Branch: `master`
- Prefix: `paper`

This syncs the Overleaf project root into local `paper/`. If Overleaf contains `utils/references/` or `utils/styles/`, those are also synced into the repo.

## 3) Push only `paper/` back to Overleaf

After committing your local changes:

```bash
utils/scripts/push_overleaf.sh
```

Defaults:
- Remote: `overleaf`
- Branch: `master`
- Prefix: `paper`

You can override them:

```bash
utils/scripts/push_overleaf.sh overleaf master paper
```

This push is non-force. The script creates a commit on top of the current Overleaf branch and makes the Overleaf root match the committed `paper/` tree from your current `HEAD`. It also includes committed files under `utils/references/` and `utils/styles/` when present.

## 4) Recommended workflow

- Keep the full project in this repo.
- Treat `paper/` as the only subtree that syncs with Overleaf.
- Keep figures generated by code under `paper/figures/` when they are needed for compilation.
- Keep raw code, data work, notes, and journal packages outside `paper/`.

## 5) Current organization after initial import

- `paper/` holds the Overleaf manuscript project and compile-time assets.
- `Notes/overleaf_imported/` holds technical notes that were previously mixed into the Overleaf project.
- `Notes/archive/` holds older drafts and archived figure material.
- `src/model/overleaf_imported/` holds model code that had previously lived inside Overleaf.
- `src/figures/scripts/` holds figure-generation scripts that had previously lived under `paper/figures/`.

If you later want Overleaf to stay clean, keep new code and notes in `src/` and `Notes/`, and only copy final paper assets into `paper/`.
